Collection

Collections are a typical UI component for listing items in a vertical line. Lets add a collection which would act as a filter on our to-do's.

Add the following markup after the navbar.

<div class="collection">
        <a href="#!" class="collection-item active">ASAP</a>
        <a href="#!" class="collection-item">Tomorrow</a>
        <a href="#!" class="collection-item">Next Week</a>
</div>

When we give a collection item the "active" class it will be highlighted.

Though we can achieve our desired layout with a flex-box. Most CSS libraries come with their own grid system which we can rely on for responsive layout. Though they may just as well be using flex-box under the hood, they provide convenient to use classes which help us achieve our desired look faster.

A CSS library would also make design decisions on our behalf such as what viewport size should be considered ‘small', ‘medium' and ‘large'.

Using classes such as s12 or m4 we can specify an element to be full width on small devices or just ⅓ of the width on medium devices. Thus we use these classes to achieve a responsive design.

We can add the following markup to only show the filters on the side for wider displays.

We've just completed the following:

  • Used section selection elements to contain our cards in a column
  • We used responsive classes s12 & m9 to give the section full width on small screens and ¾ width on wider screens
  • We also used the class container on main which added a margins on either side of its content
  • We used row and column to set the flex-direction of the elements and allow resizing with the responsive classes.

Next we want to change out the filter component for a dropdown on mobile devices. Materialize has some helper classes we can use to achieve this. We shall use the helper classes to show a dropdown instead of a collection for the filter on small devices.

Add the helper class hide-on-small-only to collection div in main and the following mark up for the dropdown after it.

<!--- Dropdown to show on mobile only -->
<div class="card teal row hide-on-med-and-up">
      <div class="input-field col s12">
        <select class="white-text">
          <option value="1">ASAP</option>
          <option value="2">Tomorrow</option>
          <option value="3">Next Week</option>
        </select>
        <label class="white-text">Filter</label>
      </div>
</div>

Materialize also needs us to add some javascript to initialize the dropdown so add this in a script tag in the bottom of the body

<script>
    document.addEventListener('DOMContentLoaded', function() {
        var elems = document.querySelectorAll('select');
        var instances = M.FormSelect.init(elems);
    });
</script>

We have used the helper classes hide-on-small-only and hide-on-med-and-up to swap out our filter component based on the screen size.

Using pseudo classes we can dynamically change the css of an element in response to an event which happens on the element, such as hover or click.

Transitions are properties we can use to customize how the effect of the styling of a pseudo class is applied.

For example we can affect the speed in which the element transitions from its original style to that of the pseudo class.

The following are the transition properties:

  • transition-property : select which property the effect is applied to eg width, height
  • transition-duration : how many seconds should a transition take
  • transition-timing-function : affects the rate of speed the effet will have eg
  • transition-delay : specifies a delay before transition begins, eg 2s, 300ms
  • linear, ease, ease-in, ease-out, step-start, others
  • transition : used to set all transition properties at once eg width 2s ease-out 200ms (property, duration, timing-function, delay)

Add the following style tag and contents to the head of the page.

<style>
  .todo {
    width: inherit;
    transition-property: width;
    transition-duration: 2s;
  }

  .todo:hover {
    width: 160px;
  }

  .todo:nth-child(1){
    transition-timing-function: ease;
  }

  .todo:nth-child(1){
    transition-timing-function: ease-in;
  }

  .todo:nth-child(1){
    transition-timing-function: linear;
  }
</style>

Go ahead and change the values to any of the following valid values and observe the result:

linear|ease|ease-in|ease-out|ease-in-out|step-start|step-end|steps(int,start|end)|cubic-bezier(n,n,n,n) where n is a number between 0 and 1

Notice: It's important to explicitly specify an initial length/width for our elements to transition from. Hence the value inherit is given to the width of our .todo selector even though materialize would have sized it according to the responsive classes. Without specifying a width the transition would not work.
